# Re:Design でできること（デザイン未経験のマーケター向け・ラフ）

デザインの専門用語やツールがわからなくても大丈夫。このシステムは「マーケの意図を形に近づける」ための補助ツールです。

---

## このシステムの目的

- **デザイナーへの発注を楽にする** … 「なんとなくこうしたい」を、具体的な言葉や画像に変える
- **デザインの良し悪しを自分でざっくり判断する** … プロ視点のフィードバックを参考にできる
- **スワイプLP・バナー・オーバーレイ用の素材を、自分で一歩まで作る** … 最終画像は外部の画像AI（NanoBanana / DALL·E など）で仕上げる想定

---

## 主な機能（大きく4つ）

### 1. プロンプトライブラリ（トップ画面）

**何ができるか**

- **お気に入りデザインを「テンプレート」として登録** … いいなと思った画像をアップロードすると、AIが「この画像を再現するための説明文（プロンプト）」を自動で作ってくれる
- **スライド用のコピーを入れると、そのテンプレートに合わせたプロンプトを生成** … 「メインコピー」「サブコピー」「追加テキスト」を入力すると、デザインを崩さずにテキストを差し替えたプロンプトが出る
- **生成したプロンプトをコピー** … そのまま NanoBanana や DALL·E などに貼って画像生成する

**デザイン未経験でも**

- 画像を選ぶだけなので、デザインの作り方は覚えなくてよい
- 社内で「このトーンで」と決めたデザインをライブラリに入れておけば、誰でも同じトーンでプロンプトを出せる

---

### 2. スライド生成（スワイプLP用）

**何ができるか**

- **LPや競合サイトのURLを入れる** … AIがそのページを分析する（3C・AIDMA などマーケの観点）
- **分析結果を見て、必要なら補足情報を入力** … 「この層向け」「この訴求を強く」など
- **AIがスライド構成を提案** … 課題提起 → ソリューション → ベネフィット → 社会的証明 → CTA の流れで、枚数とコピー案が出る
- **1枚ずつ編集・デザイン選択** … ライブラリのテンプレートから「このスライドはこのデザイン」を選べる。人物を入れない（動画オーバーレイ用）も選べる
- **各スライド用のプロンプトを自動生成** … コピーして画像AIで画像を作る

**デザイン未経験でも**

- 構成・コピーはマーケの考え方（課題→解決→ベネフィット等）で進められる
- デザインは「ライブラリから選ぶ」だけ。レイアウトや配色の知識がなくても進められる

---

### 3. ベース画像生成（オーバーレイモード）

**何ができるか**

- **参考にしたい画像（テキスト入り）を1枚アップロード** … AIが「テキストを消しただけのベース画像」を生成する
- **動画の上に載せる用のベース** … 人物や写っているシーンを除き、レイアウト・色・デザインはそのままの「背景だけ」の画像も作れる（スライド編集で「人物なし」を選んだときのプロンプトと連携）

**デザイン未経験でも**

- 「この雰囲気の背景が欲しい」「文字だけ消したい」というニーズにそのまま使える。デザインソフトは不要

---

### 4. キャラ生成（フェイススワップ・別シーン・マルチポーズ）

**何ができるか**

- **フェイススワップ** … 土台画像の顔を、別の人物写真の顔に差し替える。土台・差し替え用の画像はアップロードまたはライブラリから選択
- **キャラ別シーン** … 1枚のキャラ画像と「こういうシーン」というプロンプトを渡すと、そのキャラで別シーンの画像用プロンプトが出る
- **マルチポーズ** … 同じキャラで複数ポーズ・構図のプロンプトを一括で出し、まとめてダウンロードできる

**デザイン未経験でも**

- 画像はアップロードまたはライブラリから選ぶだけ。プロンプトは「どんなシーンにしたいか」を日本語で書けば、AIが整えた形で出てくる。実際の画像生成は外部の画像AIで行う

---

## 全体の流れ（イメージ）

1. **テンプレートをためる** … 気に入ったデザインをライブラリに登録（一度やればみんなで使える）
2. **スライドを作りたいとき** … スライド生成でURL入力 → 分析 → 構成・コピー編集 → デザイン選択 → プロンプトをコピー
3. **ベース画像が欲しいとき** … オーバーレイモードで参考画像をアップロード → テキストなし／人物なしのベースを取得
4. **キャラで複数パターン出したいとき** … キャラ生成でフェイススワップ or 別シーン or マルチポーズ → 出たプロンプトを画像AIへ

---

## 注意しておきたいこと

- **最終的な画像生成は Re:Design の外** … プロンプトをコピーして、NanoBanana・DALL·E・Flux などのサービスで画像を作る想定
- **デザインの「監修」は人間** … AIの提案はあくまでたたき台。ブランドや法規制に合わせた最終判断は人が行う
- **まずはライブラリを育てると楽** … 自社のトーンに合うデザインを少しずつ登録しておくと、スライド生成・キャラ生成がやりやすくなる

---

## 用語メモ（ラフ）

| 用語 | 意味（このシステムでのイメージ） |
|------|--------------------------------|
| プロンプト | 画像AIに「こういう画像を描いて」と伝えるための説明文。Re:Design はこれを自動生成する |
| テンプレート | 登録した参考画像＋その画像を再現するためのプロンプト。スライドの「デザインの型」として使う |
| スワイプLP | 縦スクロールでスライドをめくっていくLP。1枚1枚に課題・解決・ベネフィットなどを載せる |
| オーバーレイ | 動画の上に重ねるテキストや図。その下地になる「ベース画像」を Re:Design で作れる |
| フェイススワップ | ある画像の顔部分を、別の写真の顔に置き換えること |

---

*デザイン未経験でも、マーケの意図を「言葉とプロンプト」に落とし込めれば、デザイナーや画像AIとやりとりしやすくなります。細かい表現やルールは今後も更新していく想定です。*
